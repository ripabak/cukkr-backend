This combined skill has been split into two focused skills:

1. `add-issues-to-github`
	- Use for creating/syncing issues from local planning docs to GitHub.
2. `implement-github-issues`
	- Use for implementing existing GitHub issues and syncing progress.

Both workflows require `issues-checklist.md` to compare local issue tracking versus GitHub state.
